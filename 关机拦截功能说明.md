# TP-Link 宽带拨号 - 关机拦截功能说明

## 功能概述

程序现在支持在系统关机前自动执行断开连接并清除账号密码的操作，确保账号安全。

**主要特性：**
- ✅ 检测到关机动作时弹出警告窗口
- ✅ 提示用户需要先断开连接并清除账号
- ✅ 用户可选择立即执行清理操作
- ✅ 清理完成后允许安全关机
- ✅ 详细的日志记录和状态提示

## 实现原理

### 1. 关机事件监听

程序使用 Tkinter 的 `WM_QUERYENDSESSION` 协议来监听 Windows 系统的关机事件：

```python
self.root.protocol("WM_QUERYENDSESSION", self.on_shutdown_query)
```

### 2. 关机拦截流程

当检测到系统关机时，程序会：

1. **检查是否有保存的账号**
   - 如果有保存的账号（`self.saved_account` 不为空），则弹出警告对话框
   - 如果没有保存的账号，直接允许系统关机

2. **弹出安全警告对话框**
   - 显示警告信息，提示用户账号密码仍保存在路由器中
   - 询问用户是否立即执行断开并清除操作
   - 用户可选择"确定"（执行清理）或"取消"（跳过清理）

3. **执行断开和清除操作**（如果用户选择"确定"）
   - 启动一个非守护线程来执行断开操作
   - 等待断开操作完成（最多等待30秒）
   - 验证账号密码是否已成功清除
   - 显示清理结果（成功/失败/超时）

4. **允许系统关机**
   - 如果断开成功，显示成功消息并允许系统关机
   - 如果断开失败或超时，显示警告消息并允许系统关机
   - 如果用户选择"取消"，显示风险警告并允许系统关机

## 注意事项

### 1. 关机超时

程序最多等待30秒来完成断开操作。如果30秒内无法完成，程序会显示超时警告并允许系统继续关机。

### 2. 强制关机

如果用户强制关机（如长按电源按钮），程序可能无法完成清理操作。**强烈建议用户正常关机**，并按照警告对话框的提示操作。

### 3. 用户选择

- **推荐**：点击"是"按钮，让程序自动执行清理操作
- **不推荐**：点击"否"按钮，跳过清理操作（存在账号泄露风险）
- **最佳实践**：在使用完成后主动点击"断开连接"按钮

### 4. 网络问题

如果路由器无法访问或网络出现问题，断开操作可能会失败。程序会显示失败警告，但仍然允许系统关机。

### 5. 手动清理

如果自动清理失败，用户可以：
1. 取消关机
2. 手动点击程序中的"断开连接"按钮
3. 等待清理完成后再关机

## 技术细节

### WM_QUERYENDSESSION 消息

`WM_QUERYENDSESSION` 是 Windows 发送给应用程序的消息，询问应用程序是否可以关闭会话。

- 返回 `True`：允许系统关机
- 返回 `False`：阻止系统关机

### 用户交互设计

程序使用 Tkinter 的 `messagebox` 模块来实现用户交互：

1. **询问对话框** (`askyesno`)
   - 用于询问用户是否执行清理操作
   - 返回 `True`（用户点击"是"）或 `False`（用户点击"否"）

2. **信息对话框** (`showinfo`)
   - 用于显示清理成功的消息
   - 绿色图标，积极反馈

3. **警告对话框** (`showwarning`)
   - 用于显示清理失败、超时或用户取消的警告
   - 黄色图标，提醒用户注意

### 线程安全

程序使用线程安全的方式执行断开操作：

1. 使用 `threading.Event` 来同步等待断开完成
2. 使用非守护线程确保断开操作能够完成
3. 使用队列来安全地传递日志消息

### 异常处理

程序在断开操作中添加了异常处理，确保即使出现错误也能正确响应关机事件：

```python
try:
    result = self.run_disconnect(close_on_success=False)
    disconnect_result[0] = result
except Exception as e:
    self.log(f"❌ 断开操作出错: {e}")
finally:
    disconnect_complete.set()
```

通过实现关机拦截功能和用户交互对话框，程序现在能够在系统关机前：
