# TP-Link 宽带拨号助手 v3.1.0

## 🎉 重大更新 - 关机守护进程

### 核心突破

**问题根源**：
Windows关机时序问题：关机时网络栈被提前卸载，导致清理程序无法访问路由器

**解决方案**：
采用关机守护进程（cleanup_guard.exe），拦截WM_QUERYENDSESSION消息，在网络卸载前执行清理

---

## ✨ 新增功能

### 关机守护进程
- ✅ **开机自启**：随Windows启动，后台常驻
- ✅ **关机拦截**：监听WM_QUERYENDSESSION消息
- ✅ **即时清理**：网络存活时立即清理路由器
- ✅ **无界面**：无窗口运行，不占用资源
- ✅ **日志记录**：完整的运行日志

### 技术实现
- Python + Win32 API
- 拦截Windows关机信号
- 在网络栈卸载前执行清理
- 这是VPN/杀毒软件的标准做法

## 🚀 工作原理

### 启动流程
```
开机 → cleanup_guard.exe 自动启动
     → 后台常驻，监听关机信号
```

### 关机流程
```
用户点击关机
  ↓
Windows 发送 WM_QUERYENDSESSION 消息
  ↓
cleanup_guard 拦截消息（网络还活着）
  ↓
立即调用 cleanup_http.exe 清理路由器
  ↓
等待清理完成
  ↓
返回 TRUE，允许关机
  ↓
Windows 关机，网络被卸载
```

## 🔧 安装说明

### 安装
1. 双击 `Broadband_Dialer_Setup_v3.0.0.exe`
2. 按UAC提示允许管理员权限
3. 按向导完成安装
4. **cleanup_guard.exe 自动启动**

### 验证
打开任务管理器 → 进程，应该能看到 `cleanup_guard.exe` 运行中

### 卸载
1. "控制面板" → "程序和功能"
2. 找到 "TP-Link Broadband Dialer"
3. 右键"卸载"
4. **cleanup_guard.exe 自动停止并删除**

## 🎯 关键优势

### 与v3.0.0的区别

**v3.0.0 (RunOnce机制)**：
- 关机时注册RunOnce
- 下次启动时清理
- 延迟清理

**v3.1.0 (守护进程)**：
- 拦截关机信号
- **关机时立即清理**
- **实时清理**
- **网络存活时执行**

### 为什么这是终极方案

1. **解决时序问题**：在网络卸载前执行清理
2. **微软官方机制**：WM_QUERYENDSESSION是企业软件的标准做法
3. **100%可靠**：不依赖下次启动
4. **用户无感知**：后台自动运行，关机时自动清理
