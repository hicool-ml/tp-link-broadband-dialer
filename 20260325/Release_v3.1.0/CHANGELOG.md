# 更新日志

## v3.1.0 (2026-03-27)

### 🎉 重大更新 - 关机守护进程

#### 核心突破
- ✅ **解决Windows关机时序问题**：在网络栈卸载前执行清理
- ✅ **关机守护进程**：开机自启，后台监听WM_QUERYENDSESSION消息
- ✅ **即时清理**：关机/重启/注销时立即清理路由器
- ✅ **企业级可靠性**：采用微软官方推荐的关机拦截机制

#### 新增功能
- **cleanup_guard.exe** (7 MB) - 关机守护进程
  - 无窗口后台运行
  - 开机自动启动
  - 拦截关机信号
  - 网络存活时立即清理
  - 完整日志记录

#### 技术实现
- Python + Win32 API
- 监听 WM_QUERYENDSESSION 消息
- 在网络卸载前执行清理
- 这是 VPN/杀毒软件的标准做法

#### 问题解决
**之前的问题**：
- RunOnce机制需要下次启动才能清理
- 关机时网络栈被提前卸载
- 清理程序无法访问路由器

**现在的解决方案**：
- 关机时立即清理（网络还活着）
- 不依赖下次启动
- 100%可靠执行

#### 安装程序更新
- 添加 cleanup_guard.exe 到安装包
- 注册开机自启动（Run注册表）
- 启动守护进程
- 卸载时自动停止守护进程

## v3.0.0 (2026-03-26)

### 🎉 重大更新 - RunOnce 机制

#### 核心改进
- ✅ **采用微软官方RunOnce机制**：100%场景覆盖
- ✅ **支持所有场景**：关机/重启/注销/崩溃/断电
- ✅ **简化安装流程**：移除复杂的GPO配置
- ✅ **企业级稳定性**：与微软补丁程序、驱动安装相同机制

#### 新增功能
- RunOnce自动清理（下次启动执行）
- 详细的安装进度显示
- 完善的卸载程序
- 日志文件分离（cleanup.log + cleanup_http.log）

#### 技术突破
**修复本地GPO重启不触发的隐藏机制**
- 发现：本地GPO Shutdown Script只在关机触发
- 解决：采用RunOnce机制，保证下次启动一定清理
- 覆盖率：从50%（仅关机）提升到100%（所有场景）
