# TP-Link 宽带拨号助手 v3.1.0

## 🎉 关机守护进程 - 真正的即时清理

### 🎯 核心突破

**问题**：Windows关机时网络栈被提前卸载，导致清理程序无法访问路由器

**解决**：采用关机守护进程，拦截WM_QUERYENDSESSION消息，在网络卸载前执行清理

---

## 🔧 技术原理

### 守护进程机制

**启动时**：
```
Windows启动 → cleanup_guard.exe自动启动 → 后台常驻
```

**关机时**：
```
用户点击关机
  ↓
Windows发送WM_QUERYENDSESSION
  ↓
cleanup_guard拦截消息（网络还活着）
  ↓
立即调用cleanup_http.exe
  ↓
清理路由器账号
  ↓
返回TRUE，允许关机
  ↓
Windows关机
```

### 为什么这是唯一正确的方案

1. **时序正确**：在网络卸载前执行清理
2. **官方机制**：WM_QUERYENDSESSION是微软给企业软件的钩子
3. **100%可靠**：不依赖下次启动
4. **实时清理**：关机时立即执行

## ❓ 常见问题

### Q: cleanup_guard.exe 占用资源吗？

A: 不占用。它是无窗口程序，只在后台监听消息，CPU和内存占用极低。

### Q: 如何确认守护进程在运行？

A: 打开任务管理器 → 进程，查找 `cleanup_guard.exe`

### Q: 关机时真的清理了吗？

A: 查看日志：`C:\ProgramData\BroadbandDialer\cleanup_guard.log`

### Q: 可以禁用守护进程吗？

A: 不建议。禁用后将无法关机时清理路由器。
