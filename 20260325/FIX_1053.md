# Error 1053 修复说明

## 问题描述
Windows 服务启动时报错：
```
Error 1053: The service did not respond to the timely fashion
```

## 根本原因
Windows 要求服务必须在 **30秒内** 进入 `RUNNING` 状态，否则会报 1053 错误。

原代码在 `SvcDoRun()` 中先执行：
- `setup_logging()` - 文件IO操作
- `validate_config()` - 读取配置文件
- 最后才 `ReportServiceStatus(RUNNING)`

如果这些操作超过30秒，就会触发 1053 错误。

## 修复方案（已实现）

### ✅ 1. 先报告 RUNNING，再初始化
```python
def SvcDoRun(self):
    # ✅ 第一步：立刻报告服务已启动
    self.ReportServiceStatus(win32service.SERVICE_RUNNING)

    # ✅ 第二步：再初始化（不会触发1053）
    self.setup_logging()
    self.validate_config()
```

### ✅ 2. 延迟导入模块
```python
# ❌ 旧方式：在文件顶部导入（启动时卡顿）
# from tplink_http_cleaner import TPLinkHTTPCleaner
# from config_manager import ConfigManager

# ✅ 新方式：在使用时才导入
def _perform_cleanup(self):
    from tplink_http_cleaner import TPLinkHTTPCleaner
    from config_manager import ConfigManager
```

### ✅ 3. 修复 InstallService 参数
```python
# ❌ 旧方式
win32serviceutil.InstallService(
    None,  # 错误！
    ...
)

# ✅ 新方式
win32serviceutil.InstallService(
    pythonClassString="service_http.TPLinkCleanupServiceHTTP",
    ...
)
```

### ✅ 4. 添加 START_PENDING 状态
```python
def __init__(self, args):
    ...
    # 报告服务正在启动
    try:
        self.ReportServiceStatus(win32service.SERVICE_START_PENDING)
    except:
        pass
```

### ✅ 5. 移除 emoji 字符
避免编码问题导致服务卡死：
```python
# ❌ 旧：self.logger.info("🚨 收到系统关机通知！")
# ✅ 新：self.logger.info("[ALERT] 收到系统关机通知！")
```

## 技术要点总结

| 问题 | 解决方案 |
|------|----------|
| 启动超时 | 先 RUNNING，后初始化 |
| 模块导入慢 | 延迟导入（lazy import） |
| 安装参数错误 | 使用 pythonClassString |
| 状态报告缺失 | 添加 START_PENDING |
| 编码问题 | 移除 emoji 字符 |

## 核心原则

**服务启动阶段要做的事情越少越好！**

- ✅ 立刻报告 RUNNING
- ✅ 延迟加载重量级模块
- ✅ 文件IO放到后台
- ✅ 关键逻辑放到关机时执行
